# Issue Reporting Guide

This guide will help you submit effective bug reports and feature requests.

## Before Reporting

- Search existing issues to avoid duplicates
- Check if the issue is related to your specific setup (Factorio version, OS, etc.)
- Ensure you're using the latest version of Yarrtorio

## Bug Reports

### Use This Template

```markdown
## Bug Description

[A clear and concise description of what the bug is]

## Steps to Reproduce

1. [First step]
2. [Second step]
3. [Third step]
   ...

## Expected Behavior

[What you expected to happen]

## Actual Behavior

[What actually happened instead]

## Environment

- OS: [e.g., Windows 11, Ubuntu 22.04]
- Yarrtorio Version: [e.g., 2026.3.22]
- Factorio Version: [if applicable]

## Screenshots

[If applicable, add screenshots to help explain the problem]

## Additional Context

[Add any other context about the problem here]
```

### Tips for Good Bug Reports

- Be specific and detailed in your description
- Include all steps needed to reproduce the issue
- Mention your operating system, Yarrtorio version, and Factorio version
- Include any error messages or logs if applicable

## Feature Requests

### Use This Template

```markdown
## Feature Description

[Describe the feature you'd like to see added]

## Problem Solved

[What problem does this feature solve?]

## Proposed Solution

[How do you think this could be implemented?]

## Alternatives Considered

[Any alternative solutions you've considered?]

## Additional Context

[Add any other context or mockups about the feature request]
```

### Tips for Good Feature Requests

- Explain why this feature would be useful
- Describe the problem you're trying to solve
- Consider if this aligns with the project's goals

## Questions

For general questions about usage or setup, please:

1. Check the [README.md](README.md) for setup instructions
2. Check the [CONTRIBUTING.md](CONTRIBUTING.md) for development info
3. Search existing issues for similar questions

## Security Vulnerabilities

For security vulnerabilities, please DO NOT open a public issue. Instead, contact the maintainer directly through GitHub.

Thank you for helping improve Yarrtorio!
